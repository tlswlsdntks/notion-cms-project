# ROADMAP: 북 리뷰 사이트 (Book Reviews)

> Notion CMS 기반 개인 독서 기록 및 리뷰 공유 사이트 개발 로드맵

---

## Phase 1. 프로젝트 골격 (구조, 환경 설정)

### 왜 이 순서인가?

환경과 폴더 구조가 확정되지 않으면 이후 모든 작업이 충돌·재작업으로 이어진다.
패키지 설치, 환경변수, 디렉토리 구조를 가장 먼저 확정해야 나머지 단계가 흔들리지 않는다.

**예상 소요 시간:** 0.5일

### 작업 목록

- [ ] Next.js 15 App Router 프로젝트 생성 및 TypeScript 설정
- [ ] Tailwind CSS, shadcn/ui, Lucide React 설치 및 설정
- [ ] `@notionhq/client` 패키지 설치
- [ ] `.env.local` 생성 — `NOTION_API_KEY`, `NOTION_DATABASE_ID` 설정
- [ ] 디렉토리 구조 확정 (`app/`, `components/`, `lib/`)
- [ ] Notion 데이터베이스 생성 및 샘플 데이터 3건 입력
- [ ] `lib/notion.ts` — Notion 클라이언트 초기화 및 기본 API 연결 확인

### 완료 기준

- `npm run dev` 실행 시 오류 없이 로컬 서버 구동
- `lib/notion.ts`에서 Notion API 호출 후 샘플 데이터 3건 콘솔 출력 확인

## Phase 2. 공통 모듈 (모든 기능에서 쓰는 것들)

### 왜 이 순서인가?

핵심 기능 페이지들은 `BookCard`, `RatingStars`, `GenreFilter` 같은 공통 컴포넌트와
Notion API 함수에 의존한다. 먼저 만들어 두어야 이후 페이지 개발에서 중복 없이 재사용할 수 있다.

**예상 소요 시간:** 1일

### 작업 목록

- [ ] `lib/notion.ts` — 책 목록 조회, 단건 조회 함수 구현
  - `getBooks()` — 전체 목록 (필터/정렬 파라미터 포함)
  - `getBookBySlug(slug)` — 단건 조회
  - `getBookContent(pageId)` — Notion 페이지 본문 조회
- [ ] `components/BookCard.tsx` — 표지, 제목, 저자, 별점을 표시하는 카드 컴포넌트
- [ ] `components/RatingStars.tsx` — 1~5점 별점 표시 컴포넌트
- [ ] `components/GenreFilter.tsx` — 장르 필터 탭 컴포넌트
- [ ] 공통 타입 정의 (`types/book.ts`) — `Book`, `BookDetail` 인터페이스

### 완료 기준

- `BookCard`, `RatingStars`, `GenreFilter` 컴포넌트 단독 렌더링 확인
- `getBooks()` 호출 시 Notion 데이터 정상 반환

## Phase 3. 핵심 기능 (가장 중요한 기능)

### 왜 이 순서인가?

책 목록과 리뷰 상세 페이지가 완성되어야 서비스로서 최소한의 가치를 가진다.
통계·UX 개선보다 먼저 완성해야 MVP 검증이 가능하고, 이후 추가 기능의 방향도 결정할 수 있다.

**예상 소요 시간:** 2일

### 3-1. 홈 페이지 (`/`)

- [ ] 최근 완독 책 목록 카드 그리드 렌더링
- [ ] 연간 독서량 요약 (올해 N권) 표시
- [ ] 장르 필터 탭 연동

### 3-2. 책 목록 페이지 (`/books`)

- [ ] 전체 완독 목록 렌더링 (페이지네이션)
- [ ] 장르 / 별점 필터 동작
- [ ] 정렬 (최신순 / 별점 높은순) 동작

### 3-3. 리뷰 상세 페이지 (`/books/[slug]`)

- [ ] 책 표지, 제목, 저자, 별점 헤더 렌더링
- [ ] Notion 페이지 본문 렌더링 (`notion-to-md` 활용)
- [ ] 이전 / 다음 책 네비게이션

### 완료 기준

- 홈 → 목록 → 상세 전체 페이지 네비게이션 오류 없이 동작
- Notion에서 데이터 수정 시 페이지에 반영 확인
- 필터 적용 후 결과 정상 출력

## Phase 6. 검색 기능 (점진적 개선 1)

### 왜 이 순서인가?

책 수가 10권을 넘으면 스크롤로 찾는 것이 불편해진다. 필터는 있지만 제목·저자를 직접 입력해
찾는 검색창이 없으면 사용성이 크게 떨어진다. 가장 빈번하게 필요해지는 기능이라 1순위로 추가한다.

**MVP 선택 이유:** "특정 책을 찾고 싶은데 스크롤해야 한다" → 검색창 하나면 해결된다.

**예상 소요 시간:** 0.5일

### 작업 목록

- [ ] `/books` 페이지 상단에 검색 입력창 추가 (`SearchInput` 클라이언트 컴포넌트)
- [ ] URL searchParams `?q=검색어` 방식으로 상태 관리 (서버 컴포넌트 패턴 유지)
- [ ] `getBooks()` 함수에 `search` 파라미터 추가 — Notion 필터에 제목/저자 포함 조건 추가
- [ ] ko.json / en.json 번역 키 추가 (`books.searchPlaceholder`)

### 완료 기준

- 검색창에 제목 일부 입력 시 결과가 필터링되어 렌더링
- 검색어가 URL에 반영되어 링크 공유 가능
- 빈 결과 시 "검색 결과가 없습니다" 메시지 표시

## Phase 7. 공유 & 동적 OG 이미지 (점진적 개선 2)

### 왜 이 순서인가?

좋은 리뷰를 SNS에 공유하고 싶을 때 링크만 붙여넣으면 책 표지와 제목이 미리보기로 나와야
클릭률이 높아진다. 공유 버튼은 구현이 간단하고, OG 이미지는 Next.js App Router의
`opengraph-image.tsx` 규약으로 최소 코드로 구현 가능하다.

**MVP 선택 이유:** "리뷰 링크를 카톡에 보냈는데 미리보기가 안 나온다" → OG 태그 + 이미지 생성.

**예상 소요 시간:** 0.5일

### 작업 목록

- [ ] `/books/[slug]` 페이지에 **링크 복사** 버튼 추가 (`ShareButton` 클라이언트 컴포넌트)
- [ ] `app/[locale]/books/[slug]/opengraph-image.tsx` — 책 표지 + 제목으로 동적 OG 이미지 생성
- [ ] `generateMetadata`에 `og:image`, `twitter:card` 추가 (Phase 3에서 기본 구현, 여기서 강화)
- [ ] ko.json / en.json 번역 키 추가 (`bookDetail.copyLink`, `bookDetail.copied`)

### 완료 기준

- "링크 복사" 클릭 시 클립보드에 URL 복사, 버튼 텍스트가 "복사됨"으로 잠시 변경
- SNS 공유 시 책 표지 이미지와 제목이 미리보기에 표시
- `<head>` 소스에서 `og:image` URL 확인

## Phase 8. 독서 목표 트래킹 (점진적 개선 3)

### 왜 이 순서인가?

"올해 50권 읽기" 같은 목표를 세웠을 때 현재 몇 권 읽었는지 한눈에 보이면
동기부여가 된다. 별도 DB 없이 환경변수로 목표를 설정하고, 기존 `getBooks()` 데이터로
진행률을 계산하므로 추가 API 호출 없이 구현 가능하다.

**MVP 선택 이유:** "올해 목표를 달성하고 있는지 모르겠다" → 홈에 진행률 바 하나.

**예상 소요 시간:** 0.5일

### 작업 목록

- [ ] `.env`에 `NEXT_PUBLIC_YEARLY_GOAL=50` 추가
- [ ] `components/ReadingGoal.tsx` — 목표 권수 대비 완독 수 진행률 바 컴포넌트
- [ ] 홈 페이지 연간 독서량 요약 영역에 `ReadingGoal` 컴포넌트 삽입
- [ ] ko.json / en.json 번역 키 추가 (`home.goal`, `home.goalProgress`)

### 완료 기준

- 홈 화면에 "올해 N권 / 목표 M권 (X%)" 진행률 바 표시
- 목표 달성 시(N ≥ M) 완료 배지 또는 색상 변경
- `NEXT_PUBLIC_YEARLY_GOAL` 값 변경 시 즉시 반영

## Phase 9. 인용구 컬렉션 (점진적 개선 4)

### 왜 이 순서인가?

책에서 인상적인 구절을 기록하는 것은 독서 기록의 핵심 가치다. Notion DB에 필드 하나를
추가하고 별도 페이지로 모아 보여주면 사이트의 차별점이 생긴다. 앞의 Phase들보다 구현 범위가
크지만, 기존 Notion 연동 코드를 재사용할 수 있어 예상 이상으로 빠르다.

**MVP 선택 이유:** "기억에 남는 구절을 한 곳에서 보고 싶다" → `/quotes` 페이지.

**예상 소요 시간:** 1일

### 작업 목록

- [ ] Notion DB에 `인용구` 필드 추가 (rich_text, 여러 줄 허용)
- [ ] `types/book.ts`의 `Book` 인터페이스에 `quote?: string` 추가
- [ ] `lib/notion.ts`의 `pageToBook()`에 인용구 매핑 추가
- [ ] `app/[locale]/quotes/page.tsx` — 인용구가 있는 책 목록, 카드 형태 표시
- [ ] `i18n/routing.ts`에 `/quotes` 경로 추가
- [ ] ko.json / en.json 번역 키 추가 (`nav.quotes`, `quotes.title`, `quotes.empty`)

### 완료 기준

- `/quotes` 페이지에 인용구가 있는 책의 구절이 카드로 표시
- 인용구 카드 클릭 시 해당 책 상세 페이지로 이동
- Notion에서 인용구 추가/수정 시 60초 이내 반영
